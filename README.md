# 🧠 Psychology Wiki

A **curated, LLM-maintained knowledge base focused exclusively on Psychology** — spanning clinical psychology, cognitive science, social psychology, neuroscience, and related fields.

This wiki is designed to collect, organise, and interconnect psychological research, theories, and concepts. Rather than manually summarising papers and articles, it relies on autonomous AI workflows to ingest new source material, build conceptual connections across the knowledge graph, and retrieve synthesised knowledge on demand.

> **Language Note:** All wiki-generated content (summaries, concept pages, index entries) is written in **Traditional Chinese (繁體中文)**. Raw source files in `raw/` are preserved as-is in their original language.

## Core Architecture

This repository maintains strict separation between **immutable source files** and the **dynamic knowledge graph** generated and managed by the LLM.

- **`raw/`**: The immutable storage directory. Drop your psychology source files (PDFs, Markdown articles, transcripts) here. The LLM will strictly read these files and *never* modify or delete them.
- **`wiki/`**: Structured directory generated by the LLM containing:
  - **`summaries/`**: Detailed summary pages for each ingested raw document.
  - **`concepts/`**: Extracted granular entities and distinct concepts (e.g., theorists, frameworks, phenomena) linking back to relevant sources.
- **`index.md`**: The central catalogue where the LLM records all entities, concepts, and source summaries. It serves as the entry point for navigating the synthesised knowledge.
- **`log.md`**: The chronological ingestion log that tracks what the LLM has processed and added to the wiki.
- **`AGENTS.md`**: The foundational schema and configuration rulebook that instructs the LLM on how to manage, maintain, and structure the wiki.
- **`.agents/`**: Hidden directories containing the workflows and skills (e.g., `ingest`, `query`, `lint`) that power the AI agent's operations.

## How to Use this Wiki

The entire premise of this project is to interact with it via an LLM. Here are the primary workflows you can trigger:

### 1. Ingesting New Knowledge (`ingest`)

1. Drop a new psychology source file (e.g., a research paper, article, or transcript) into the `raw/` folder.
2. Ask the LLM to process it by invoking the skill:

   ```text
   Use /ingest to process the new file in the raw folder
   ```

3. **What happens?** The AI reads the source, saves a Traditional Chinese summary into `wiki/summaries/`, extracts psychological concepts into individual Markdown files within `wiki/concepts/`, reviews and updates existing concept files when relevant, updates `index.md` with links, and appends the run to `log.md`.

### 2. Querying and Synthesis (`query`)

Ask the LLM a complex psychology-related question using the query skill:

```text
Use /query to answer: What do the ingested articles say about cognitive dissonance?
```

- **What happens?** The AI reads `index.md` to identify relevant pages, reads those individual markdown pages, and synthesises a well-researched answer supported entirely by your knowledge base. It can even generate derived outputs like comparison tables or thematic analyses if requested.

### 3. Linting and Maintenance (`lint`)

Ask the LLM to keep the wiki healthy by invoking the linting skill:

```text
Use /lint to health-check the wiki
```

- **What happens?** The AI will actively scan for orphan pages, flag contradictory information, ensure cross-references are linked correctly, and optionally use web searches to fill in missing gaps or suggest new concept pages.

## Built With

This project relies on AI agent workflows (as documented in `AGENTS.md` and the existing Skills) and can seamlessly integrate with standard Markdown-based knowledge management systems like Obsidian. Local settings for such external editors are ignored via `.gitignore` to maintain a clean repository.
